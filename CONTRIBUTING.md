# Contributing

Thanks for your interest in contributing.

## How to contribute

1. Open an issue first for bugs or feature ideas.
2. Create a focused branch per change.
3. Keep pull requests small and easy to review.
4. Update documentation when behavior changes.
5. Add a changelog entry for user-visible changes.

## Coding guidelines

- Use clear names and keep functions focused.
- Prefer safe defaults and explicit error handling.
- Keep scripts compatible with Windows PowerShell 5.1 unless explicitly documented otherwise.

## Pull request checklist

- [ ] Script runs locally.
- [ ] No parser errors.
- [ ] Documentation updated if needed.
- [ ] Changelog updated for user-visible behavior.
